# V2 Product Brief

This file describes the intended buyer experience for the `v2` Telegram bot.

## Buyer Progression

`OnlyFans Verified -> Starter -> Plus -> Pro`

Important:

- `OnlyFans Verified` is the post-verification access state.
- It is not a paid Telegram product tier.
- Users should keep seeing the tier ladder clearly.

## Tone

- Private premium shop
- Clear and structured
- Personal but low-pressure
- Concise copy
- Lower-case product buttons
- Friendly emojis are welcome

Avoid:

- aggressive sales copy
- shaming language
- corporate phrasing

## Main Menu

- `starter unlock 🗝️`
- `plus menu 💎`
- `pro menu 🔥`
- `my access 👤`
- `my purchases 📦`
- `rules & boundaries ⚠️`

## Locked Progression

Higher-tier areas should be visible before access, but locked items should not expose locked prices.

The locked feeling should create progression and curiosity, not rejection.

## Verified State

Verified users should land in the private menu and understand:

- they are in
- Starter is the first paid step
- Plus and Pro exist
- PPVs become cheaper as they progress

Suggested copy:

`you’re verified ✅`

`you’re inside the private menu now.`

`start with Starter to open the first unlock.`

`Plus and Pro are visible too, but they open as you move through the tiers. as you progress, PPV-prices decrease by 25% per tier. buy more, save more :D`

## Starter

Product:

- `starter unlock` - `$37`

Includes:

- 3 random pictures
- 2 from `dickpic_hard`
- 1 from `dickpic_soft`
- instant delivery

Starter buttons:

- `unlock starter 🗝️`
- `what comes next 👀`
- `view my access 👤`
- `back`

## Plus

Menu:

1. Best Value Bundle
2. Premium PPVs
3. Back

Best Value Bundle:

- `$97`
- 1 `stroking`
- 1 `strip`
- Starter buying this upgrades directly to `Pro`

Premium PPVs:

- `stroking` - `$67`
- `ass_noboxers` - `$67`
- `strip` - `$67`

Tier progression:

- Starter buying one individual `$67` PPV becomes `Plus`
- Plus buying another `$67` PPV becomes `Pro`

## Pro

Products:

- `5–10 min jerkoff + cum` - `$170`
- `voice note` - `$80`
- `chat access`

Manual fulfillment:

- Pro products should create admin fulfillment requests
- no custom request review flow

## Discount Rule

Each higher tier gets `25% off` lower-tier PPVs only.

Interpretation confirmed by user:

- discounts apply downward, not upward
- locked-tier prices should stay hidden before access

## My Purchases

The buyer should be able to see:

- their tier
- completed purchases
- pending manual items
- delivered items

## Test Build Rule

During `v2` development, the test bot should be effectively admin-only:

- only the owner/admin should be able to experience flows
- all testing commands should be foolproof
- a second BotFather token will be used for `v2`
